# CodeTown — 应用流程文档

**Version 1.2** | 基于 PRD v6.5 | 2026年3月

本文档定义 CodeTown 中的每个屏幕、导航路径、用户流程、决策点、成功状态和错误状态。AI 开发者不应猜测用户如何在应用中移动——请严格遵循本文档。

---

## 1. 屏幕清单

> **注意：** 本项目为单页应用（SPA），不使用 `react-router-dom` 等重型路由库。内部导航通过 React state 控制，分享链接（`/p/{slug}`、`/u/{name}`）使用 `wouter@3.3.5` 处理路由解析。

| 屏幕 ID | 屏幕名称 | 路由/状态映射 | 描述 | 需要登录 |
|---------|----------|------|------|----------|
| `MAP` | 小镇地图 | `/` | 首页，4栋建筑 + 中央广场 | 是 |
| `DESK` | 设计桌 | `/level/{n}/desk` | 左侧文件夹 + 中央蓝图画布 | 是 |
| `SITE` | 工地 | `/level/{n}/site` | 左侧 Agent 团队 + 右侧 Mirror 思考流 | 是 |
| `PROJECT_VIEW` | 项目查看 | `/p/{slug}` | 已完成项目的公开 iframe 展示 | 否 |
| `TOWN_VIEW` | 小镇查看 | `/u/{name}` | 公开只读小镇地图 | 否 |
| `SETTINGS` | 设置面板 | 模态框 | 音量、API Key 管理、登出 | 是 |
| `BYOK_MODAL` | 建筑许可证 | 模态框 | API Key 输入表单 | 否 |
| `AUTH_MODAL` | 注册/登录 | 模态框 | 邮箱/手机号+密码注册登录 | 否 |

## 2. 应用入口

### 2.1 主入口 — codetown.app/

**触发条件：** 用户在浏览器打开 codetown.app

**流程序列：**
1. 应用加载
2. 检查用户是否已登录（检查浏览器本地是否存有有效的登录凭证）
3. 未登录 → 显示 `AUTH_MODAL`，必须登录后才能进入地图
4. 已登录 → 从数据库加载用户 `town_state`
5. 渲染 `MAP` 屏幕，显示 4 栋建筑

**成功状态：**
- 地图显示正确的建筑状态（灰色/彩色，基于进度）
- L1 建筑可点击（已解锁）
- L2/L3/L4 建筑显示 🔒（如前置关卡未完成）

**错误状态：**
- 加载用户数据网络错误 → 显示地图使用本地/默认状态 + 黄色横幅"无法加载存档，请刷新"

---

### 2.2 项目直链 — codetown.app/p/{slug}

**触发条件：** 用户打开分享的项目链接

**流程序列：**
1. 根据 `share_slug` 从数据库获取项目
2. 找到 → 渲染 `PROJECT_VIEW`，iframe 展示项目
3. 未找到 → 显示 404 页面

**成功状态：**
- 全屏 iframe 展示项目
- 底部栏显示"在 CodeTown 上建造"CTA 按钮
- "返回"按钮（有账号则回到自己的地图；否则跳转营销页）

**错误状态：**
- 项目不存在 → "这个项目不存在或已被删除" + "去 CodeTown 看看"按钮

---

### 2.3 小镇直链 — codetown.app/u/{name}

**触发条件：** 用户打开分享的小镇链接

**流程序列：**
1. 根据 `display_name` 从数据库获取用户
2. 找到 → 渲染 `TOWN_VIEW`，只读地图
3. 未找到 → 显示 404 页面

**成功状态：**
- 只读地图显示所有已完成（彩色）建筑
- 点击彩色建筑 → 在 `PROJECT_VIEW` 中打开该项目
- 灰色建筑可见但不可点击
- 底部栏显示"在 CodeTown 上建造"CTA

**错误状态：**
- 用户不存在 → "这个小镇不存在" + "去 CodeTown 看看"按钮

## 3. 核心用户流程

### 3.1 流程：首次用户完整旅程（L1）

**触发条件：** 新用户首次打开 codetown.app

**前置条件：** 用户已完成登录（进入小镇即需登录，无游客模式）

**流程序列：**

```

├── 用户打开 codetown.app
├── 检测到未登录 → 弹出 AUTH_MODAL
├── 用户注册/登录（邮箱/手机号+密码）
└── 登录成功 → 进入 MAP

步骤 2: MAP（初始状态）
├── 用户看到 4 栋灰色建筑
├── L1（面包店）= 已解锁，可点击
├── L2/L3/L4 = 锁定，显示 🔒
└── 用户点击 L1 建筑

步骤 3: BYOK_MODAL
├── 模态框出现："建筑许可证"
├── 显示获取 API key 的说明
├── API key 输入框
├── 用户粘贴 key
├── 决策点：Key 格式检查
│   ├── 格式不符 → 红色错误"这不像一个有效的密钥"，保持模态框
│   └── 格式正确 → 继续
├── 将 key 存储到 React state + sessionStorage
└── 关闭模态框 → 导航到 DESK

步骤 4: DESK（L1 设计桌）
├── 左侧边栏：3 个文件夹已解锁（委托单/装修方案/需求清单），6 个文件夹锁定
├── 中央：蓝图，3 个必填卡槽（脉冲轮廓）

├── L1 模板预填：
│   ├── 委托单："面包店新品展示页"，项目类型："网站"
│   ├── 装修方案：4 个预设缩略图供选择
│   └── 需求清单：3 个复选框 + 1 个自定义槽
```

### 3.2 流程：BYOK（建筑许可证）

**触发条件：** 用户点击任何已解锁建筑，但没有存储的 API key

**流程序列：**

```
1. 用户在 MAP 上点击建筑
2. 检查：sessionStorage/React state 中是否有 API key？
   ├── 是 → 跳过，直接进入 DESK
   └── 否 → 显示 BYOK_MODAL

3. BYOK_MODAL 显示：
   ├── 标题："建筑许可证"
   ├── 副标题："需要一把钥匙才能开始建造"
   ├── 说明（可折叠）：
   │   ├── "请家长帮忙注册模型服务商账号"
   │   ├── "创建 API 密钥"
   │   └── "粘贴到下方"
   ├── 输入框（密码类型，placeholder："粘贴你的 API Key"）
   ├── 确认按钮
   └── 背景变暗，不可交互

4. 用户粘贴并确认
5. 格式验证：
   ├── 空 → "请输入密钥"
   └── 不以 `sk-` 开头 → "这不像一个有效的密钥"
   └── 以 `sk-` 开头 → 通过

6. 验证通过后：
   ├── 存储到 React state
   ├── 存储到 sessionStorage
   ├── 关闭模态框
   └── 导航到该关卡的 DESK
```

**成功状态：**
- Key 存储在内存和 sessionStorage 中
- 用户进入设计桌
- 页面刷新后 Key 仍保留（同一标签页会话内）
- 标签页关闭时 Key 被清除

**错误状态：**
- 输入为空 → 输入框下方红色文字，模态框保持打开

**注意：** 此处仅做 `sk-` 前缀格式校验（Opencode API Key）。无效 key 将在构建开始时失败（见第 5 节 ERR-1）。

### 3.3 流程：注册/登录

**触发条件：**
- 用户首次打开 codetown.app（未登录）
- 用户在任意页面登录状态过期

**流程序列：**

```
1. 触发事件发生
2. 显示 AUTH_MODAL（强制，不可绕过）

3. AUTH_MODAL 显示：
   ├── 标题："注册后保存你的小镇"
   ├── Tab 切换：「邮箱登录」/「手机号登录」
   ├── 邮箱模式：邮箱输入框 + 密码输入框
   ├── 手机号模式：手机号输入框 + 密码输入框
   ├── "进入小镇"按钮（登录）/ "注册账号"按钮
   └── "已有账号？登录"链接（切换模式）

4. 用户输入凭证，点击确认
5. 后端调用 Supabase Auth 注册/登录
6. 成功 → 返回 access_token + refresh_token + user 数据
7. 前端存储 token，进入 MAP
```

**成功状态：**
- 用户已登录
- Session 在浏览器关闭后仍保留（refresh_token）
- Town state 从数据库加载，进入地图

**错误状态：**
- 邮箱格式无效 → "请输入有效的邮箱地址"
- 手机号格式无效 → "请输入有效的手机号"
- 密码过短 → "密码至少 8 位"
- 凭证错误 → "邮箱/手机号或密码错误"
- 网络错误 → "网络错误，请稍后重试"

### 3.4 流程：设计桌（L2/L3/L4）

**触发条件：** 用户从 MAP 进入某个关卡（L2、L3 或 L4）

**前置条件：** 用户有 API key，前置关卡已完成

**流程序列：**

```
1. 导航到 /level/{n}/desk
2. 从数据库加载草稿（如存在）
3. 渲染 DESK 屏幕：
   ├── 左侧边栏：根据关卡解锁状态显示文件夹
   ├── 中央：带卡槽的蓝图（必填/可选/锁定）
   ├── Tab：[🗒️ 设计桌（高亮）] [🏗️ 工地（灰色）]
   └── 预填模板内容（因关卡而异）

4. 用户交互循环：
   ├── 点击文件夹 → 展开显示卡片选项
   ├── 从文件夹拖动卡片到蓝图 →
   │   ├── 匹配的卡槽 → 高亮 + 吸附就位
   │   └── 不匹配的卡槽 → 无高亮，卡片弹回
   ├── 编辑卡片内容 → 自动保存到 drafts 表
   └── 移除卡片 → 拖出卡槽或点击 X

5. 检查必填卡槽是否已填：
   ├── 任一必填为空 → "📐 制定工作计划"按钮灰色
   └── 所有必填已填 → 按钮启用

6. 用户点击"📐 制定工作计划"
7. Meta Planner 运行：
   ├── L2+：调用 LLM 组织卡片
   ├── 卡片动画排列成从上到下的顺序
   └── 按钮变为"🚀 开始建造"

8. 用户点击"🚀 开始建造"
9. 切换到 SITE Tab
10. 构建开始（见流程 3.5）
```

**各关卡解锁状态：**

| 关卡 | 必填卡槽 | 可选卡槽 | 锁定卡槽 |
|------|----------|----------|----------|
| L1 | 委托单, 装修方案, 需求清单 | （无） | 工匠, 技能包, 规则, 流程控制, Plug, 图书馆 |

### 3.6 流程：测试失败与重试

**触发条件：** Tester Agent 报告某需求未满足

**流程序列：**

```
1. Tester 左侧卡片显示 ❌
2. 右侧 Mirror 追加气泡："❌「{需求}」好像不对。再试。"
3. 检查该需求的重试次数：
   ├── 次数 < 3 → 自动重试
   │   ├── Builder 左侧卡片重新激活（⟳）
   │   ├── Builder 尝试修复
   │   ├── Tester 重新验证
   │   └── 循环直到通过或次数 = 3
   └── 次数 = 3 → 暂停等待用户决策

4. 达到最大重试次数时：
   ├── 构建暂停
   ├── 模态框显示三个选项：
   │   ├── "再试一次" → 重置重试计数，继续
   │   ├── "跳过这个任务" → 标记为跳过，继续
   │   └── "停止构建" → 中止，返回 DESK（只读状态清除）
```

## 5. 错误流程

### 5.1 API Key 错误（ERR-1）

**触发条件：** Key 被 API 服务商返回 401/429/服务错误

**时机：** 构建开始或 Agent 执行期间

**流程序列：**
```
1. API 调用失败并返回错误
2. 构建暂停
3. 模态框出现："密钥无效，请重新输入"
4. 用户输入新 key
5. 非空验证通过 → 关闭模态框，重试失败的操作
6. 构建从暂停处继续
```

**关键点：** 构建不会终止。用户可恢复。

---

### 5.2 Agent 超时（ERR-2）

**触发条件：** 单次 Agent 调用超过 60 秒

**流程序列：**
```
1. 检测到超时
2. 自动重试一次（2 秒延迟）
3. 如仍失败：
   ├── 左侧 Agent 卡片摇晃（CSS 动画）
   ├── 显示"重试"和"跳过"按钮
   └── 用户决定
```

---

### 5.3 空/格式错误响应（ERR-3）

**触发条件：** Agent 返回空内容或无法解析的格式

**流程序列：**
```
与 ERR-2 相同：自动重试一次，然后用户决定
```

---

### 5.4 测试者最大重试次数（ERR-4）

**触发条件：** 测试者对同一需求失败 3 次

**流程序列：**
```
1. 第 3 次失败
2. 构建暂停
3. 模态框显示 3 个选项：
   ├── "再试一次" → 重置计数器，Builder 重试
   ├── "跳过这个任务" → 标记跳过，继续
   └── "停止构建" → 中止全部，返回 MAP
```

---

### 5.5 输出过大（ERR-5）

**触发条件：** 生成的 HTML 超过 200KB

**流程序列：**
```
1. 大小检查失败
2. Toast："项目太大了，已简化"
3. 截断至 200KB
4. 用截断版本继续
```

---

### 5.6 MetaPlanner 失败（ERR-6、ERR-7）

**L1：**
```
用户无感知 — 静默降级为硬编码顺序
```

**L2+：**
```
1. 自动重试一次
2. 如仍失败：
   ├── 显示"重试"和"停止"按钮
   └── 用户决定
```

---

### 5.7 WebSocket 断开（ERR-8、ERR-9）

**流程序列：**
```
1. WebSocket 意外关闭
2. 黄色横幅出现："重连中…"
3. 已完成工位状态保留
4. 指数退避重试（5 次尝试）
5. 如重连成功 → 从最后检查点继续
6. 如 5 次全部失败：
   ├── 横幅："网络不稳定，请刷新重试"
   └── 构建进度丢失（设计桌草稿保留）
```

---

### 5.8 并行分支失败（ERR-10）

**流程序列：**
```
1. 失败分支的左侧 Agent 卡片显示 ❌
2. 另一分支独立继续
3. 在合并点：
   ├── 失败分支视为"跳过"
   └── 构建用成功分支的输出继续
```

---

### 5.9 Plug 失败（ERR-12、ERR-13）

**预制 Plug 失败：**
```
右侧 Mirror 追加气泡："外部服务暂时不可用"
左侧 Agent 卡片显示"重试"/"跳过"按钮
```

**魔法连接无匹配：**
```
Agent 收到："暂时不支持这种连接"
Agent 尝试替代方案或通过输出告知用户
```

---

### 5.10 规则死循环（ERR-14）

**流程序列：**
```
1. 系统检测到同一循环的第 3 次迭代
2. 强制停止
3. Toast："规则循环了，已自动停止"
4. 构建继续，不再执行循环规则
```

---

### 5.11 图书馆限制

**单条粘贴 >2000 字符：**
```
Toast："这条资料太长了，上限 2000 字"
截断或拒绝（实现选择）
```

**总计 >8000 tokens：**
```
Toast："资料太多了，已取前 8000 token"
```

**文件 >500KB：**
```
拒绝上传
Toast："文件太大了，上限 500KB"
```

**文件解析失败：**
```
Toast："这个文件打不开，试试粘贴文本？"
```

**产品化后：** 切换为千问 API，降低成本。代码层封装 `MODEL_PROVIDER` 配置变量，切换时只改一处。

---

### 6.2 已登录用户状态

**存储位置：** Supabase（Postgres）

**数据表：**
- `users`：个人资料、current_level、town_state
- `projects`：每个已完成项目
- `drafts`：自动保存的设计桌内容
- `skills`：用户创建的技能
- `rules`：用户创建的规则
- `library_refs`：上传的参考文件

**同步：**
- 关键操作时实时同步（保存项目、完成关卡）
- 卡片编辑时自动保存草稿（防抖）

---

### 6.3 中断恢复

| 场景 | 重新进入时 |
|------|------------|
| DESK 编辑中退出 | 从 `drafts` 表恢复草稿 |
| SITE 构建中退出 | DESK Tab 激活，卡片恢复，需重新点击"开始建造"（进度丢失） |
| 揭幕后退出 | 项目已持久化，建筑已变彩色 |
| 浏览器崩溃 | 与退出相同 — 构建进度丢失，草稿保留 |

## 9. 路由表

> **路由实现说明：**
> 1. **内部导航**（如 `/` ↔ `/level/1/desk`）通过 React 顶层组件的状态（如 `currentView`）切换实现，配合 `window.history.pushState` 更新 URL，不引入 `react-router-dom`。
> 2. **分享链接路由**（`/p/{slug}`、`/u/{name}`）使用 `wouter`（v3.3.5）处理外部直接访问时的路由解析，无需完整路由库。
> 3. 所有内部页面均需登录，未登录时强制弹出 AUTH_MODAL。

| 路由/状态映射 | 组件 | 认证 | 描述 |
|------|------|------|------|
| `/` | MapScreen | 必须 | 主小镇地图 |
| `/level/1/desk` | DeskScreen | 必须 | L1 设计桌 |
| `/level/1/site` | SiteScreen | 必须 | L1 工地 |
| `/level/2/desk` | DeskScreen | 必须 | L2 设计桌 |
| `/level/2/site` | SiteScreen | 必须 | L2 工地 |
| `/level/3/desk` | DeskScreen | 必须 | L3 设计桌 |
| `/level/3/site` | SiteScreen | 必须 | L3 工地 |
| `/level/4/desk` | DeskScreen | 必须 | L4 设计桌 |
| `/level/4/site` | SiteScreen | 必须 | L4 工地 |
| `/p/{slug}` | ProjectViewScreen | 无 | 公开项目查看 |
| `/u/{name}` | TownViewScreen | 无 | 公开小镇查看 |

除公开分享链接外，所有内部视图均需登录，未登录时强制弹出 AUTH_MODAL。
